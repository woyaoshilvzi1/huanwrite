# User Workflow Checklist

## 目标

定义项目所有者在工作台中完成一篇小说生产和投稿准备的完整流程。

## 全局流程

- [ ] 用户能进入工作台并看到当前全部稿件。
- [ ] 用户能创建题材。
- [ ] 用户能筛选题材。
- [ ] 用户能把题材推进到写前规划。
- [ ] 用户能确认规划是否可执行。
- [ ] 用户能基于已确认规划生成正文候选。
- [ ] 用户能查看候选内容和候选来源。
- [ ] 用户能对候选发起审稿。
- [ ] 用户能查看审稿结论。
- [ ] 用户能按审稿结论返修候选。
- [ ] 用户能人工确认候选合入正文。
- [ ] 用户能查看正式正文版本。
- [ ] 用户能生成投稿包候选。
- [ ] 用户能确认投稿包。
- [ ] 用户能记录真实投递。
- [ ] 用户能记录平台回执。
- [ ] 用户能记录确认过的收入或回款。
- [ ] 用户能选择正式稿件进入短剧改编。
- [ ] 用户能确认短剧改编稿。
- [ ] 用户能为短剧改编稿生成投稿包。

## 用户判断点

- [ ] 题材进入规划前，用户能看到卖点、冲突、平台、体量和风险。
- [ ] 规划确认前，用户能看到故事承诺、人物关系、结构、文风和禁用规则。
- [ ] 候选合入前，用户能看到候选正文、审稿结论、来源和合入影响。
- [ ] 投稿前，用户能看到标题、简介、标签、平台备注和投前检查。
- [ ] 记录投递前，用户能确认这是真实投递事实。
- [ ] 记录收入前，用户能确认这是真实收入或回款事实。

## 中断与继续

- [ ] 用户能看到每篇稿件当前状态。
- [ ] 用户能看到每篇稿件下一步建议。
- [ ] 用户能看到每篇稿件当前风险。
- [ ] 用户能看到每篇稿件最近动作。
- [ ] 用户能从任一状态继续工作，而不依赖聊天记录。

## 完成验收

- [ ] 从题材到投稿包的主流程可以被完整走通。
- [ ] 从正式稿件到短剧改编投稿包的主流程可以被完整走通。
- [ ] 任一步骤的结果都有明确落点。
- [ ] 任一步骤的状态都能被继续接手。
