# Observability Checklist

## 目标

定义运行证据如何记录，方便接力、排查和复盘。

## 运行记录

- [ ] 每个动作有动作类型。
- [ ] 每个动作有关联稿件。
- [ ] 每个动作有开始时间。
- [ ] 每个动作有结束时间或失败时间。
- [ ] 每个动作有输入摘要。
- [ ] 每个动作有输出位置。
- [ ] 每个动作有状态。
- [ ] 每个动作有错误信息或完成摘要。

## 证据边界

- [ ] 运行证据不替代正文。
- [ ] 运行证据不替代规划。
- [ ] 运行证据不替代审稿结论。
- [ ] 运行证据不保存敏感凭据。
- [ ] 运行证据服务排查，不替用户做最终判断。

## 接力信息

- [ ] 当前项目状态可被读取。
- [ ] 当前稿件状态可被读取。
- [ ] 当前下一步动作可被读取。
- [ ] 当前风险和待确认项可被读取。

## 完成验收

- [ ] 每次动作都有可追溯记录。
- [ ] 接手者能判断项目当前状态。
- [ ] 接手者不用翻聊天记录才能继续。
