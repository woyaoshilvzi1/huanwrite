# Provider And Secrets Checklist

## 目标

定义模型接入和敏感信息边界。

## Provider

- [ ] 模型服务配置有明确来源。
- [ ] 模型调用能区分创作、审稿、平台判断等用途。
- [ ] 模型调用有超时设置。
- [ ] 模型调用失败有可读错误。
- [ ] 模型调用结果进入候选或运行证据。

## Secrets

- [ ] API key 不写入项目文件。
- [ ] API key 不写入稿件。
- [ ] API key 不写入运行日志。
- [ ] API key 不写入工作台状态。
- [ ] 平台账号密码不写入项目。
- [ ] cookie 不写入项目。
- [ ] 合同隐私不写入项目。
- [ ] 身份证、银行卡、税务隐私不写入项目。

## 完成验收

- [ ] 项目文件中没有敏感凭据。
- [ ] 模型配置可被验证。
- [ ] 模型输出不会绕过候选流程。
