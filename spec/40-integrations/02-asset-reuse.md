# Asset Reuse Checklist

## 目标

定义成熟资产如何成为当前项目正式能力，避免重复造轮子。

## 复用原则

- [ ] 能直接复用的写作规则直接纳入当前项目。
- [ ] 能直接复用的题材判断规则直接纳入当前项目。
- [ ] 能直接复用的质量 checklist 直接纳入当前项目。
- [ ] 能直接复用的语料索引结构直接纳入当前项目。
- [ ] 能直接复用的动作流程直接纳入当前项目。
- [ ] 能直接复用的数据契约思想直接纳入当前项目。
- [ ] 复用后成为当前项目正式资产。

## 正式资产落点

- [ ] 写作规则进入 `.huanwrite/assets/writing-rules/`。
- [ ] 质量规则进入 `.huanwrite/assets/quality-rules/`。
- [ ] 题材规则进入 `.huanwrite/assets/topic-rules/`。
- [ ] 平台规则进入 `.huanwrite/platforms/`。
- [ ] 语料摘要进入 `.huanwrite/assets/corpus/`。
- [ ] 模板进入 `.huanwrite/assets/templates/`。

## 复用边界

- [ ] 正式项目不运行时读取参考路径。
- [ ] 正式项目不 import 参考代码。
- [ ] 正式项目不要求参考目录存在。
- [ ] 正式项目不把参考材料名称写成业务概念。
- [ ] 正式项目只保留已经进入当前结构的资产。

## 复用验收

- [ ] 每个复用资产有当前项目落点。
- [ ] 每个复用资产有当前项目用途。
- [ ] 每个复用资产有来源备注。
- [ ] 删除参考材料后，当前项目仍能运行和测试。
- [ ] 当前项目文档只描述正式资产，不描述参考材料。
