# Workbench

## 职责

工作台负责把题材、规划、正文、候选、审稿、投稿包、短剧改编、平台雷达、运行记录和任务状态变成可操作界面。

工作台不是业务真相源。工作台只读写 API 暴露的正式状态。

## 页面能力

- 看板总览。
- 多稿线状态列。
- 搜索和状态筛选。
- 拖拽改状态。
- 题材创建和选择。
- 结构化规划编辑。
- 规划动作入口。
- 正文编辑。
- 质量门、负责人、备注。
- 候选审稿和合入。
- 工作台动作面板。
- 输出查看器。
- 后台任务列表和停止请求。
- 平台雷达。
- 参考资产。

## 技术边界

- 前端使用 React/Vite。
- 图表使用 ECharts。
- 后台任务更新使用 WebSocket。
- API 调用集中在 `packages/web/src/api`。
- 页面状态聚合集中在 controller hook。
- 业务规则不写进 UI 组件。

## 验收

- E2E 能从 UI 创建题材、规划、正文、候选、审稿、合入和投稿包。
- E2E 能运行全部工作台动作。
- E2E 能验证真实 run、job、trace 和状态变化。
- E2E 能验证图表真实渲染。
- E2E 能验证拖拽改状态。
- E2E 能验证运行输出正文可读。
