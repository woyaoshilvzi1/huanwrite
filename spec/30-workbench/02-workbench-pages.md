# Workbench Pages Checklist

## 目标

定义工作台必须有哪些页面，以及每个页面负责什么。

## Dashboard

- [ ] 展示全部稿件。
- [ ] 展示每篇稿件状态。
- [ ] 展示每篇稿件下一步动作。
- [ ] 展示当前风险和待确认项。
- [ ] 不编辑正文。
- [ ] 不替代具体工作页面。

## Topics Page

- [ ] 创建题材。
- [ ] 编辑题材判断。
- [ ] 标记题材状态。
- [ ] 将题材推进到规划。
- [ ] 不生成正式正文。

## Planning Page

- [ ] 展示规划内容。
- [ ] 编辑规划内容。
- [ ] 检查规划完整度。
- [ ] 确认规划。
- [ ] 修改已确认规划后标记为需要重新确认。
- [ ] 不直接合入正文。

## Manuscript Page

- [ ] 展示正式正文。
- [ ] 编辑正式正文。
- [ ] 展示正文统计。
- [ ] 展示正文变更来源。
- [ ] 不展示未经确认的候选为正式正文。

## Candidates Page

- [ ] 展示候选列表。
- [ ] 展示候选详情。
- [ ] 展示候选来源。
- [ ] 发起候选审稿。
- [ ] 标记候选废弃。
- [ ] 不绕过审稿合入正文。

## Review Page

- [ ] 展示审稿结果。
- [ ] 展示问题清单。
- [ ] 展示修改建议。
- [ ] 发起返修动作。
- [ ] 标记人工判断。
- [ ] 不直接修改正文。

## Submission Page

- [ ] 展示投稿包。
- [ ] 编辑投稿包。
- [ ] 确认投稿包。
- [ ] 记录真实投递。
- [ ] 记录平台回执。
- [ ] 不保存账号密码或后台隐私。

## Adaptation Page

- [ ] 展示短剧改编状态。
- [ ] 展示 10 集一卡。
- [ ] 展示人物小传。
- [ ] 发起改编审稿。
- [ ] 确认改编稿。
- [ ] 不覆盖原小说正文。

## Runs Page

- [ ] 展示动作记录。
- [ ] 展示运行状态。
- [ ] 展示错误信息。
- [ ] 展示输出位置。
- [ ] 不替代业务真相源。

## Settings Page

- [ ] 展示模型配置来源。
- [ ] 展示安全边界说明。
- [ ] 检查连接状态。
- [ ] 不把 API key 写入项目文件。

## 完成验收

- [ ] 每个页面只有一个主职责。
- [ ] 每个页面能说清不负责什么。
- [ ] 页面写入只进入对应真相源。
- [ ] 页面能力覆盖完整用户流程。
- [ ] 页面组件只负责展示和触发用户意图。
- [ ] 页面控制逻辑只负责调用 API、刷新状态和报告结果。
- [ ] 页面统计和图表数据整理独立于图表渲染。
- [ ] 集中管理的契约不能变成 UI、API、业务规则混写的上帝文件。
- [ ] 分文件维护的模块不能把同一职责拆散到多个随机位置。
