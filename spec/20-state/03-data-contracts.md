# Data Contracts Checklist

## 目标

定义核心资料对象必须具备的信息。

## Topic

- [ ] 有唯一 ID。
- [ ] 有标题。
- [ ] 有一句话想法。
- [ ] 有核心卖点。
- [ ] 有主角困境。
- [ ] 有主冲突。
- [ ] 有读者爽点。
- [ ] 有目标平台。
- [ ] 有目标体量。
- [ ] 有风险备注。
- [ ] 有状态。
- [ ] 有创建时间和更新时间。

## Plan

- [ ] 有唯一 ID。
- [ ] 关联 Topic。
- [ ] 关联稿件。
- [ ] 有故事承诺。
- [ ] 有主角目标。
- [ ] 有主冲突。
- [ ] 有人物关系。
- [ ] 有结构卡片。
- [ ] 有文风规则。
- [ ] 有禁用句式。
- [ ] 有疲劳词。
- [ ] 有目标平台。
- [ ] 有状态。
- [ ] 有确认时间。

## Manuscript

- [ ] 有唯一 ID。
- [ ] 有标题。
- [ ] 关联 Topic。
- [ ] 关联当前 Plan。
- [ ] 有正文路径。
- [ ] 有字数统计。
- [ ] 有状态。
- [ ] 有最新变更来源。
- [ ] 有创建时间和更新时间。

## Candidate

- [ ] 有唯一 ID。
- [ ] 关联 Manuscript。
- [ ] 关联 Plan。
- [ ] 有动作类型。
- [ ] 有输入摘要。
- [ ] 有输出路径。
- [ ] 有状态。
- [ ] 有生成时间。
- [ ] 有审稿关联。

## Review

- [ ] 有唯一 ID。
- [ ] 关联 Candidate。
- [ ] 有审稿依据。
- [ ] 有结论。
- [ ] 有问题清单。
- [ ] 有修改建议。
- [ ] 有风险等级。
- [ ] 有创建时间。

## SubmissionPackage

- [ ] 有唯一 ID。
- [ ] 关联 Manuscript 或 Adaptation。
- [ ] 有目标平台。
- [ ] 有标题方案。
- [ ] 有短简介。
- [ ] 有长简介。
- [ ] 有标签。
- [ ] 有平台备注。
- [ ] 有投前检查项。
- [ ] 有状态。

## SubmissionRecord

- [ ] 有唯一 ID。
- [ ] 关联 SubmissionPackage。
- [ ] 有平台。
- [ ] 有投递时间。
- [ ] 有投递方式。
- [ ] 有回执状态。
- [ ] 有下一步动作。
- [ ] 有备注。

## RevenueRecord

- [ ] 有唯一 ID。
- [ ] 关联 Manuscript、Adaptation 或 SubmissionRecord。
- [ ] 有收入类型。
- [ ] 有金额。
- [ ] 有确认日期。
- [ ] 有状态。
- [ ] 有备注。
- [ ] 不包含银行卡、身份证或税务隐私。

## 完成验收

- [ ] 每类资料都有唯一 ID。
- [ ] 每类资料都有关联关系。
- [ ] 每类资料都有状态。
- [ ] 每类资料都有时间记录。
- [ ] 敏感信息不进入数据契约。
