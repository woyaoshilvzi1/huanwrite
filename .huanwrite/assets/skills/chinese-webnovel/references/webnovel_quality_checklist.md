# 网文质量检查清单

## 章内结构

- [ ] 本章开头前几段有抓手，不是平铺日常
- [ ] 本章中段至少推进剧情、关系、设定理解中的一项
- [ ] 到后段时，局面已经和开头不同
- [ ] 章末停在变化发生的那一拍

## 开头检查

- [ ] 前 300-500 字有抓手
- [ ] 不是天气/日常/背景介绍开场
- [ ] 主角尽快出场
- [ ] 卖点或危机尽快显形

## 黄金三章检查

- [ ] 第 1 章有明确事件
- [ ] 第 2 章冲突升级或卖点展开
- [ ] 第 3 章建立继续追读的理由

## 本章推进检查

- [ ] 本章有核心事件
- [ ] 本章局面发生变化
- [ ] 本章不是纯过渡水章
- [ ] 至少推进剧情、关系、设定理解中的一项

## 信息投放检查

- [ ] 没有大段背景直灌
- [ ] 信息尽量跟着冲突出现
- [ ] 读者不需要一口气记住太多设定

## 场景检查

- [ ] 每场戏有目标
- [ ] 每场戏有阻碍
- [ ] 每场戏有变化
- [ ] 场景里有人在做事，不是原地说感受
- [ ] 没有可以整段删除却不影响剧情的部分

## 章末检查

- [ ] 章末停在变化上
- [ ] 章末有危机/决定/发现/反转之一
- [ ] 章末不是总结式收尾
- [ ] 章末能自然拉到下一章

## 语言检查

- [ ] 没有空泛抒情堆砌
- [ ] 没有连续大段同义情绪重复
- [ ] 句子顺滑，读起来不卡
- [ ] 对话符合人物身份
- [ ] 关键情绪尽量用动作、反应、对白落地

## 容易水的地方

- [ ] 没有大段背景解释
- [ ] 没有角色重复表达同一个情绪
- [ ] 没有连续几段都不给新信息或新变化
- [ ] 没把本章写成只过桥、不落事的过渡章

## 连载检查

- [ ] 没忘掉前文已经给出的承诺
- [ ] 没突然引入过多新设定
- [ ] 伏笔有推进，不是只埋不收
- [ ] 当前剧情仍然服务于故事引擎
